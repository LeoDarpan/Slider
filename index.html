This is how you make a Slider using Vanila JavaScript !

Let's see how to make it. Study the HTML carefully and get ready to do the math. Its purely the magic of the math empowering the CSS and JS to do this.

Making full page sliders is fairly easy. You can find a great deal of videos out there with people teaching the same.

But the slider with multiple slides in the view and which pushes slides from left to right or vice-versa, one by one, is something that is quite challenging to make.

Now, there are already pre-built carousels available for you to download, not to mention, the famous Owl carousel, but making it one by yourself is something far more satisfying.

So, I decided to give it a go and trust me, it was easy but demanded calculative smartness.

CSS:

Here is the math trick that I played here:

  1. Calculate the ultimate width of one slide.
    This means the normal width + the margin on left and right.
    So here it becomes 150px + 10px + 10px = 170px.
  2. Next thing to do is to decide the total number of slides that we need in the slider.
    In my case it is 7, so the width should be: 1190px.
    But you may notice it as 1210px. Can you guess why? Notice the padding that I have given above in the code?
    Naturally, it also adds to the width, so 20px extra.
  3. Next, we need to put this slider in a container, which could restrict the visibility of the slider to the number of slides we want it to show.
    So, I put it in a window that shows only three slides at a time with a width of 530px, again with 20px additional to the normal 510px width.
  4. That's it fellas. That's all the math. Now the next trick is to tune JS.

.main{ padding: 10px; height: auto; width: 600px; margin: 0 auto; transform: scale(0); transition: all .9s ease-in-out; @include flex; .window{ height: auto; width: 530px; overflow: hidden; .slider{ padding: 10px; width: 1210px; height: 170px; transition: all .5s ease-in-out; .slide{ margin: 0 10px; padding: 0; display: block; float: left; width: 150px; height: 150px; img{ width: 100%; max-width: 100%; height: inherit; } } } } .left, .right{ div{ width: 30px; height: 4px; background-color: #000; } } .left{ .first{ transform: rotate(-45deg); position: relative; top: -10px; } .second{ transform: rotate(45deg); position: relative; top: 6px; } } .right{ .first{ transform: rotate(-135deg); position: relative; top: -10px; } .second{ transform: rotate(135deg); position: relative; top: 6px; } } .left:hover, .right:hover{ cursor: pointer; .first, .second{ background-color: #ffa500; } } }
  1. We need to make such a logic that slides the slider, as the name suggests. For that, we can employ transform property.
  2. For every click, on the left or rigth custom made arrows on far ends, the slide should slide left or right.
  3. The translateX property from the group of transform properties could do the trick.
    Hence, on every click, I move the slider by the offset equal to the ultimate width of one slide that we calculated above (170px).
  4. To bring the slider to the initial position, we need to monitor the total clicks that take it the last slide plus one more click to bring to the original position.
    That's why I am counting till 850px (which equals five clicks) on the right custom made arrow.
    Not to mention, clicking on the left arrow moves the slider by putting negative values in the translateX property which you can notice yourself.
  5. That's it folks, with such a clever trick, you can make your own slider using HTML, CSS and pure JS, without needing any library for your project to do that.

JS:

var left = document.querySelectorAll(".left"); var slider = document.querySelector(".slider"); var right = document.querySelectorAll(".right"); left[0].addEventListener("click", ()=>{ offset += 170; if(offset > 0) offset = 0; translateX1(offset); }); right[0].addEventListener("click", ()=>{ offset -= 170; if(offset <= -850) offset = 0; translateX1(offset); }); function translateX1(offset){ slider.style.transform = "translateX("+offset+"px)"; }

With a sad heart though, this slider is plagued with some challenges to overcome.

  1. The width needs to stay fixed, which means it can't be alloted in percentage yet.
  2. New slides can't be freely added without tweaking some calculations.
    So, the developer will be handling a lot of tweak calls, which makes it less viable to be sold.
  3. Its, not responsive, which is a need of the hour nowadays. The slider is responsive now!!!
